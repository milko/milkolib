Class DataSource

Data source or URL object.

This class implements a data source, that is, an URL that contains the connection properties needed by classes representing connection instances, such as servers and databases.

When instantiated, this class will parse the provided connection URL and store the connection parameters in the object's properties, the supported offsets are:

  • Milko\PHPLib\DataSource::PROT: The protocol.
  • Milko\PHPLib\DataSource::HOST: The host.
  • Milko\PHPLib\DataSource::PORT: The port.
  • Milko\PHPLib\DataSource::USER: The user name.
  • Milko\PHPLib\DataSource::PASS: The user password.
  • Milko\PHPLib\DataSource::PATH: The path.
  • Milko\PHPLib\DataSource::QUERY: The query parameters.
  • Milko\PHPLib\DataSource::FRAG: The fragment.

There is also a set of member accessor methods that can be used to manage the data source elements:

  • Milko\PHPLib\DataSource::Protocol(): Manages the protocol.
  • Milko\PHPLib\DataSource::Host(): Manages the host.
  • Milko\PHPLib\DataSource::Port(): Manages the port.
  • Milko\PHPLib\DataSource::User(): Manages the user name.
  • Milko\PHPLib\DataSource::Password(): Manages the user password.
  • Milko\PHPLib\DataSource::Path(): Manages the path.
  • Milko\PHPLib\DataSource::Query(): Manages the query.
  • Milko\PHPLib\DataSource::Fragment(): Manages the fragment.

Casting the object to a string will return its data source name.

There is no value checking when setting properties, this is the responsibility of the caller: to ensure the URL is valid instantiate a new object.

# Path( mixed $theValue = NULL , boolean $asArray,… )

Manage data source path.

Manage data source path.

This method will manage the data source path, provide NULL to retrieve the current value, FALSE to delete the path, or any other type to set it.

This method features a second parameter that can be used to retrieve the path value as an array featuring the different elements of the path:

  • NULL: The method will return the path string.
  • TRUE: The method will split the path using the DIRECTORY_SEPARATOR token.
  • other: The provided value will be converted to a string and the method will split the path using the provided token.

The method will return the old value when deleting and the current value in all other cases.

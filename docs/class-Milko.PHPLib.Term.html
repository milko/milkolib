Class Term

Term object.

This class implements an object that represents a concept or term, it is the ancestor of all objects involved in the data dictionary and ontology.

Objects of this class feature the following properties:

  • kTAG_NS: This represents the term namespace, it is a document handle (Collection::NewDocumentHandle()) that references the term which is the current term's namespace. This property is optional.
  • kTAG_LID: This represents the term local identifier, it is a string code that uniquely identifies the term among all terms belonging to its namespace. This property is required.
  • kTAG_GID: This represents the term global identifier, it is a string code that uniquely identifies the term among all terms in all namespaces. This value is computed by default by concatenating the namespace's term (kTAG_NS) global identifier with the current term's local identifier (kTAG_LID), separated by the kTOKEN_NAMESPACE_SEPARATOR token. This property is required.
  • kTAG_NAME: This represents the term name or label, it is a short description or human readable identifier that can be used to label or name the term. This property is required.
  • kTAG_DESCRIPTION: This represents the term description or definition, it is a text which describes in detail what the term represents and, in the case of the term, may be considered its definition. This property is optional.

The term's Collection::OffsetKey() property is set by extracting the MD5 checksum of its kTAG_GID global identifier

The Container::ClassOffset() property is automatically set in the constructor and should not be changed.

Finally, the RevisionOffset() property is managed directly by the database driver.

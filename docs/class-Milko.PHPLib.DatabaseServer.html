Class DatabaseServer

Database server ancestor object.

This abstract class is the ancestor of all classes representing database server instances.

This class uses its inherited Milko\PHPLib\Server class to handle the server connection and implements a public standard interface to handle database specific operations and resources.

As with its ancestor, the actual implementation of the database operations is delegated to a protected interface that must be implemented by derived concrete classes, this is why this class is declared abstract.

When instantiating this class a default database and collection can be provided in the DataSource::Path() part of the connection string, these will be added to the server and all operations will use these unless explicitly indicated.

This class features a generalised public interface for communicating with the server:

  • Milko\PHPLib\DatabaseServer::Databases(): Return the list of databases on server.
  • Database(): Set, retrieve or reset database object.
  • Milko\PHPLib\DatabaseServer::DropDatabase(): Drop a database.

The above public methods do not implement the actual operations, this is delegated to a protected virtual interface which must be implemented by derived concrete classes.

# Databases( string $theFlags = Milko\PHPLib\Server::kFLAG_DEFAULT, array $theOptions = NULL )

Return server databases list.

Return server databases list.

This method can be used to retrieve the list of server databases, the method features the following parameters:

  • $theFlags: A bitfield providing the following options:
    • Milko\PHPLib\Server::kFLAG_CONNECT: If set, the server will connect if necessary, if not set and the server is not connected, the method will raise an exception.
    • Milko\PHPLib\Server::kFLAG_NATIVE: If not set, the method will return an array of database names, if set, the method will return the result from the native driver.
  • $theOptions: An optional list of options to be provided to the native driver.

# ListCollections( string $theDatabase = NULL, boolean $doConnect = FALSE, boolean $doNative = FALSE )

List database collections.

List database collections.

This method can be used to retrieve the list of database collections, the method features the following parameters:

  • $theDatabase: The database name or NULL for the default database collections list.
  • $doConnect: If TRUE the method will attempt to connect to the server if not already connected.
  • $doNative: If TRUE the method will return the native result returned by the server, if not, the method will return a list of names as an array. If set to false and the server is not connected, the method will raise an exception.

Manage default database.

Manage default database.

This method can be used to set, retrieve or reset the default server database, the method features the following parameters:

  • $theValue: The database name or the operation:
    • NULL: Retrieve the current default server database.
    • FALSE: Reset the current default server database, will reset also the default collection, if set.
    • string: Set the default database by name.
  • $doConnect: Assert server connection (only relevant if setting or resetting):
    • TRUE: Connect if necessary.
    • FALSE: Raise exception if not connected.

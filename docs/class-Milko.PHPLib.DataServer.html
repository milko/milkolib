Class DataServer

Data server ancestor object.

This abstract class is the ancestor of all classes representing data server instances.

A data server is essentially a database server, it is divided into databases, which in turn are divided into collections that contain data records.

This class uses its inherited Milko\PHPLib\Server class to handle the server connection and implements a public standard interface to handle database operations and resources.

The class implements a common public interface that manages Milko\PHPLib\Database objects:

  • Milko\PHPLib\DataServer::ListDatabases(): Return the list of database names on the server.
  • Milko\PHPLib\DataServer::WorkingDatabases(): Return the list of working database names.
  • Milko\PHPLib\DataServer::RetrieveDatabase(): Create or retrieve a database object.
  • Milko\PHPLib\DataServer::ForgetDatabase(): Clear or dispose of a database object.
  • Milko\PHPLib\DataServer::DropDatabase(): Drop a database.

Most of the above methods feature a bitfield parameter that determines the actions to take in the event a database is not found or the connection to the server is not open:

  • Milko\PHPLib\Server::kFLAG_CONNECT: If set, the server connection will be automatically opened if necessary.
  • Milko\PHPLib\Server::kFLAG_CREATE: If set and the indicated database was not found, the database will be created in the process.
  • Milko\PHPLib\Server::kFLAG_ASSERT: If set, an exception will be raised if the connection is not open and the Milko\PHPLib\Server::kFLAG_CONNECT switch is off and if the indicated database cannot be found and the Milko\PHPLib\Server::kFLAG_CREATE flag is not set.

The public methods do not implement the actual operations, this is delegated to a protected virtual interface which must be implemented by derived concrete classes:

  • Milko\PHPLib\DataServer::databaseList(): Return the list of server database names.
  • Milko\PHPLib\DataServer::databaseCreate(): Create and return a Milko\PHPLib\Database object.
  • Milko\PHPLib\DataServer::databaseRetrieve(): Return a Milko\PHPLib\Database object.

Each time a database object is retrieved, the object will store it in the Milko\PHPLib\DataServer::$mDatabases data member for fast subsequent retrievals.

# RetrieveDatabase( string $theDatabase, string $theFlags = self::kFLAG_DEFAULT, mixed $theOptions = NULL )

Return a database object.

Return a database object.

This method can be used to create or retrieve a database object, it features the following parameters:

  • $theDatabase: The database name.
  • $theFlags: A bitfield providing the following options:
    • Milko\PHPLib\Server::kFLAG_CONNECT: If set, the server will connect if necessary, if not set and the server is not connected, the method will return NULL.
    • Milko\PHPLib\Server::kFLAG_CREATE: If set, the method will create the database if it doesn't already exist.
    • Milko\PHPLib\Server::kFLAG_ASSERT: If set, the method will raise an exception if the server is not connected or if the database cannot be found.
  • $theOptions: An array of options representing driver native options.

The method will either return a Milko\PHPLib\Database object, or NULL if the server is not connected or if the database was not found.

# ForgetDatabase( string $theDatabase )

Dispose of a database object.

Dispose of a database object.

This method can be used to dispose of a database object contained in the working databases collection.

If the provided database name doesn't match any entries among the working databases, the method will return NULL, if it does, the method will return the related Milko\PHPLib\Database object.

The method features a single parameter representing the database name.

Working databases.

Each time a database is requested this list will be searched first by default.

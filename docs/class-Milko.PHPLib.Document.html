Class Document

Document ancestor object.

This class is essentially a Milko\PHPLib\Container that keeps track of its class name, it is the ancestor of all persistent classes, thus includes the global tokens ('tokens.inc.php'}, and global tags ('tags.inc.php').

The class constructor will set in its Collection::ClassOffset() property the name of its current class. This must be taken into consideration, because when instantiating an object derived from this class from the contents of another derived object, the class property will be overwritten, which means that the object should be replaced.

For the purpose of persisting, the document features a set of offsets that contain information regarding identifiers, class and revisions:

  • Collection::KeyOffset(): The offset of the document unique key.
  • Collection::ClassOffset(): The offset of the document class.
  • Collection::RevisionOffset(): The offset of the document revision.

To set the document key you would do

$document[ $collection->KeyOffset() ] = $key;

and to retrieve the key,

$key = $document[ $collection->KeyOffset() ];

These offsets are declared by the enclosing collection, which is also responsible of instantiating (Collection::NewDocument() and serialising Collection::NewNativeDocument() the document: this is because the above property tags depend on the native database engine and may also depend on the business logic of the collection.

The class also features a method, Milko\PHPLib\Document::Validate(), which should check whether the object has all the required attributes and is fit to be stored in the database, if that is not the case, the method should raise an exception; this method must be called before storing documents in the database. In this class we assume the object is valid.

Documents store their status in a data member, a public interface is provided to set and probe a series of states:

  • Persistent state: This state indicates whether the the document is stored or was retrieved from a collection, only Milko\PHPLib\Collection objects may set this state, probing the state is public.
    • Milko\PHPLib\Document::SetPersistentState(): Set the persistent state; requires providing the $this of the instance that is setting the state.
    • Milko\PHPLib\Document::GetPersistentState(): Get the persistent state.
  • Modification state: This state indicates whether the document was modified since it was instantiated, only the current object or a Milko\PHPLib\Collection instance may set this state, probing the state is public.
    • Milko\PHPLib\Document::SetModificationState(): Set the modification state.
    • Milko\PHPLib\Document::GetModificationState(): Get the modification state.

A special public method, Milko\PHPLib\Document::SetKey(), is used by Milko\PHPLib\Collection instances to set the document key (@link Collection::JeyOffset()}) after it was inserted, this is useful when the key was not provided and the database generated one automatically; this method will reset the modification state (IsModified()) after setting the key, so that the object will result clean after being inserted, this method can be called only by Milko\PHPLib\Collection instances.

Set the document modified state.
